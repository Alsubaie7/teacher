# مستند تصميم: إعادة هيكلة منصة المعلم

**التاريخ:** 2026-07-11
**الحالة:** معتمد للتخطيط
**النطاق:** إعادة بناء شاملة للواجهة والهيكلة (Information Architecture + Visual Design + Navigation)

---

## 1. الخلفية والهدف

منصة المعلم نظام إدارة صف دراسي (PWA عربي RTL) مبني بـ HTML/CSS/JS خام + Supabase (مصادقة + مزامنة). المنصة غنية بالمميزات لكن هيكلتها الحالية تعاني من تشتت وازدواج يجعلها معقّدة وغير احترافية.

**الهدف:** إعادة هيكلة المنصة لتكون **سهلة واحترافية**، عبر:
- توحيد التنقل حول "الفصل كوحدة مركزية".
- استبدال ثيم الإكسل بنظام تصميم احترافي نظيف.
- إبراز الميزات المدفونة (الكتاب، السبورة، العروض).
- الحفاظ الكامل على كل الوظائف والبيانات الحالية.

**غير قابل للمساس:** منطق الأعمال، بنية بيانات Supabase (`profiles`, `user_data`), المصادقة والاشتراكات، وكل الوظائف الموجودة. هذا **إعادة تصميم للواجهة والهيكلة**، وليس إعادة كتابة للمنطق.

---

## 2. المبادئ التوجيهية

1. **الفصل هو الوحدة المركزية** — المعلم يفكّر بـ"فصلي" ثم "وش أسوي فيه".
2. **مكان واحد لكل شيء** — لا ازدواج، لا ثلاث طرق لنفس الإجراء.
3. **الاستباقية** — الرئيسية تعرض "وش يحتاج انتباهك" لا مجرد بيانات.
4. **الاحترافية بالنظافة** — مساحات، تسلسل بصري، ألوان هادفة (لا زخرفة).
5. **حفظ الوظائف** — كل ميزة موجودة تبقى، لكن في مكانها المنطقي.

## 3. نظام التصميم البصري (نيلي عصري)

### الألوان
| الدور | اللون |
|------|------|
| الأساسي (Primary/Ink) | `#4f46e5` نيلي |
| الأساسي الفاتح (خلفيات/شارات) | `#eceafe` |
| خلفية الصفحة | `#f5f6fb` |
| السطح (كروت) | `#ffffff` |
| الحدود | `#e7e8f0` |
| النص الأساسي | `#1f2340` |
| النص الثانوي | `#7a7f9e` |
| نجاح/حاضر | `#16a34a` (خلفية `#e7f6ec`) |
| تحذير/متأخر | `#d97706` (خلفية `#fdf0dc`) |
| خطر/غائب | `#dc2626` (خلفية `#fdeaea`) |
| ألوان الفصول (تمييز) | نيلي، تركوازي `#0ea5a3`، كهرماني `#e0a020`… (يختارها المعلم) |

- **الوضع الافتراضي: فاتح.** يوجد زر تبديل للوضع الليلي (متغيّرات CSS لكلا الوضعين).
- **الخط:** Tajawal (كما هو).

### الأساسيات (Tokens)
- زوايا: `11px` للعناصر، `14–16px` للكروت.
- حدود: `1px solid var(--line)`.
- ظلال: ناعمة جداً وقليلة (`0 10px 30px rgba(...,.06)` للكروت المرتفعة فقط).
- مسافات: إيقاع `rem` رأسي (1rem / 1.5rem)، `px` داخلي (8/12/16).

### المكونات الأساسية (تُبنى مرة وتُعاد)
- الشريط الجانبي (nav item عادي/نشط).
- بطاقة إحصائية (stat card).
- كرت لوحة (panel) برأس + رابط "عرض الكل".
- شارة (pill/chip) بألوان الأدوار.
- زرّ مقسّم (segmented control) للحضور.
- جدول بيانات نظيف (رأس رمادي فاتح، صفوف بحدود خفيفة).
- صورة رمزية بالأحرف (initials avatar).
- شريط تبويبات (tabs) للفصل.

## 4. بنية المعلومات والتنقل

### المستوى ١ — التنقل الرئيسي (٤ عناصر فقط)
على الكمبيوتر: **شريط جانبي** (يمين في RTL). على الجوال: **تنقل سفلي**.
```
▦ الرئيسية    ▤ فصولي    ▥ الجدول    ◔ التحليلات
```
أسفل الشريط الجانبي: `⚙ الإعدادات` · `◉ الحساب` (قائمة: الملف، تصدير/استيراد، الإحالات، لوحة الإدارة للمشرف، تسجيل الخروج).

### الشريط العلوي (مبسّط)
`تحية + سياق اليوم` (يمين) · `بحث سريع (⌘K)` + `وضع ليلي` + `مساعد AI` (يسار). أُزيل ازدحام الأزرار (حجم الخط ينتقل للإعدادات).

### المستوى ٢ — مساحة عمل الفصل (Class Workspace)
عند دخول أي فصل، شريط تبويبات موحّد لكل ما يخصّه، مع مسار تنقّل (breadcrumb) وزر **"تبديل الفصل"**:
```
✓ الحضور والسلوك   ★ الدرجات   ◍ المجموعات   ▤ الكتاب والعروض   ◉ الطلاب
```
هذا يُلغي:
- ازدواج "الرئيسية/الفصول".
- تشتت الحضور/الدرجات/المجموعات عبر ٣ أماكن.
- غموض "الفصل النشط" (الآن أنت صراحةً داخل فصل محدّد).

### خريطة التنقل
```
الرئيسية (مركز اليوم)
فصولي ──▶ [فصل] ──▶ الحضور والسلوك | الدرجات | المجموعات | الكتاب والعروض | الطلاب
                         └─▶ ملف الطالب
الجدول
التحليلات
الحساب ──▶ الإعدادات | الإحالات | تصدير/استيراد | الإدارة (مشرف) | خروج
```

## 5. الصفحات — قبل / بعد

### 5.1 الرئيسية → "مركز اليوم"
**قبل:** Hero حسب الوقت + ٤ بطاقات + كروت الفصول (نسخة مكررة من صفحة الفصول).
**بعد:**
- تحية + سياق ("لديك ٣ حصص اليوم").
- **بطاقة "الحصة الآن"** بارزة (نيلية): الفصل الحالي + الوقت + الدرس، مع زرّين مباشرين: **افتح الكتاب** + **سجّل الحضور**.
- ٤ بطاقات إحصائية نظيفة (حصص اليوم، حضور اليوم، فصولي، إجمالي الطلاب).
- لوحة **"فصولي"** مختصرة (قائمة صفوف، لا كروت كبيرة) + نسبة حضور لكل فصل.
- لوحة **"يحتاج انتباهك"** (طلاب تعدّوا الغياب، درجات ناقصة، تقرير AI جاهز).

### 5.2 فصولي
**قبل:** كروت إكسل + أزرار مشاركة/استيراد.
**بعد:** شبكة كروت فصول نظيفة (لون الفصل، اسمه، مادته، عدد الطلاب، نسبة حضور، زر دخول). أزرار "إضافة فصل / استيراد بكود / مشاركة" في شريط علوي للصفحة. **الكرت يصبح بوابة لمساحة عمل الفصل.**

### 5.3 مساحة عمل الفصل
تبويبات موحّدة (القسم ٤). كل تبويب:
- **الحضور والسلوك:** جدول احترافي (القسم ٦) — منتقي تاريخ، ملخّص بشارات، زرّ حضور مقسّم لكل طالب، شارات سلوك قابلة للإضافة.
- **الدرجات:** جدول درجات نظيف + بطاقات إحصائية (المعدل، أعلى، أدنى، ناجح، لم يُدخل) + فلاتر + مخطط.
- **المجموعات:** سحب وإفلات، ليدربورد، جلسات (كما هي وظيفياً، بأسلوب جديد).
- **الكتاب والعروض:** تصفّح الكتاب التفاعلي + فتح السبورة الذكية + العروض المرتبطة بالدروس. (إبراز الميزات المدفونة.)
- **الطلاب:** قائمة/جدول طلاب الفصل + إضافة/استيراد جماعي + دخول ملف الطالب.

### 5.4 الجدول
جدول أسبوعي (٥ أيام × حصص) بأسلوب نظيف؛ خلايا فارغة بزر `+`، خلايا معبّأة بلون الفصل. أزرار: تعديل الأوقات، طباعة، مسح.

### 5.5 التحليلات
**قبل:** بطاقات ورسوم كلها بثيم إكسل (رأس أخضر + خلايا).
**بعد:** بطاقات إحصائية نظيفة + رسوم بيانية حديثة (اتجاه الحضور، توزيع التقديرات، مقارنة الفصول، طلاب يحتاجون اهتمام). ألوان الأدوار.

### 5.6 ملف الطالب
ترويسة (صورة رمزية، اسم، فصل) + بطاقات: الحضور، السلوك، الدرجات، الإحالات، ملاحظات AI. أسلوب "سجل بيانات" (data record) نظيف.

### 5.7 الإحالات / الإعدادات / الإدارة
تُنقل تحت قائمة الحساب. جداول ونماذج بأسلوب التصميم الجديد. الوظائف كما هي.

## 6. استبدال ثيم الإكسل

يُستبدل ثيم الإكسل (رأس أخضر، أعمدة A/B/C، صيغ `FALSE=()`) بجداول احترافية في كل مكان:
| العنصر | قبل (إكسل) | بعد (احترافي) |
|--------|-----------|---------------|
| رأس الجدول | أعمدة A/B/C + صيغة | عناوين واضحة (الطالب، الحضور، السلوك) |
| الحضور | خلية | زرّ مقسّم ملوّن (حاضر/متأخر/غائب) |
| السلوك | إيموجي في خلايا | شارات نظيفة قابلة للإضافة (`+`) |
| الطالب | نص | صورة رمزية + اسم |
| الملخّص | صف إكسل | شارات ملوّنة أعلى الجدول |
| البطاقات/التحليلات | رأس إكسل أخضر | بطاقات ورسوم نظيفة |

**يُحافظ على:** فكرة تتبّع السلوك (تفاعل، نسي الكتاب…) — تُعرض كشارات بدل خلايا الإكسل. الحفظ التلقائي يبقى.

## 7. النهج التقني

- **الحفاظ على المعمارية القائمة** (نمط كائنات `Pages`, `Router`, `DB`, `AI`… في JS خام) — لأنها تعمل وتتزامن مع Supabase. إعادة الهيكلة تطال:
  - **`index.html`:** استبدال الشريطين العلويين بشريط جانبي + تنقل سفلي متجاوب.
  - **`Router` + `Pages`:** إعادة تنظيم المسارات حول مساحة عمل الفصل (توحيد classDetail والتبويبات)، إزالة ازدواج dashboard/classes.
  - **`style.css`:** إعادة كتابة كنظام تصميم جديد قائم على متغيّرات CSS (فاتح/داكن)، حذف ثيم الإكسل.
  - دوال العرض (render) لكل صفحة: إخراج HTML جديد يستهلك المكوّنات الجديدة، مع إبقاء نفس مصادر البيانات ومعالجات الأحداث.
- **تقسيم الملفات (اختياري، مرحلة لاحقة):** يمكن تقسيم `app.js` (6286 سطر) و`style.css` إلى وحدات لتسهيل الصيانة — لكنه ليس شرطاً لنجاح إعادة الهيكلة، ويُؤجَّل حتى تستقر الواجهة.
- **PWA / service worker / manifest:** يبقى؛ يُحدّث `theme-color` للنيلي وتُرفع نسخة الكاش.

## 8. المراحل (كل مرحلة = خطة تنفيذ مستقلة)

1. **نظام التصميم + القشرة (Shell):** متغيّرات CSS الجديدة، الشريط الجانبي/السفلي المتجاوب، الشريط العلوي، الوضع الفاتح/الداكن. (بدون تغيير محتوى الصفحات بعد.)
2. **الرئيسية "مركز اليوم":** بطاقة الحصة الآن، الإحصائيات، فصولي المختصرة، يحتاج انتباهك.
3. **فصولي + مساحة عمل الفصل:** الكروت الجديدة، التبويبات الموحّدة، تبديل الفصل، breadcrumb.
4. **الحضور والسلوك (استبدال الإكسل):** الجدول الاحترافي، الحضور المقسّم، شارات السلوك.
5. **الدرجات:** الجدول والبطاقات والمخطط بالأسلوب الجديد.
6. **المجموعات + الكتاب والعروض + الطلاب.**
7. **الجدول + التحليلات.**
8. **الإحالات + الإعدادات + الإدارة + ملف الطالب.**
9. **تلميع نهائي + اختبار متجاوب (جوال/كمبيوتر) + الوضع الليلي.**

> ملاحظة: كل مرحلة يجب أن تُبقي المنصة قابلة للاستخدام دون كسر الوظائف أو المزامنة.

---

## 9. خارج النطاق

- تغيير منطق الأعمال أو بنية بيانات Supabase.
- إضافة ميزات جديدة (هذا إعادة هيكلة، لا توسعة).
- تغيير المصادقة/الاشتراكات.
- تقسيم الكود إلى وحدات (مؤجّل، اختياري).

---

## 10. معايير النجاح

- التنقل الرئيسي ٤ عناصر واضحة، والفصل مساحة عمل موحّدة.
- لا ازدواج بين الرئيسية والفصول.
- الكتاب/السبورة/العروض يُوصل لها بنقرة من الرئيسية أو تبويب الفصل.
- لا أثر لثيم الإكسل؛ جداول احترافية نظيفة.
- كل الوظائف والبيانات الحالية تعمل كما كانت.
- متجاوب على الجوال والكمبيوتر، فاتح وداكن.
